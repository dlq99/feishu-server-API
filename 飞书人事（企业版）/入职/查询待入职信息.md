## 查询待入职

该接口用于根据待入职人员 ID(支持批量)查询待入职人员信息，信息包含姓名、手机号等个人信息和任职信息。
- 延迟说明：数据库主从延迟 2s 以内，即：直接创建待入职后2s内调用此接口可能查询不到数据。
- 性能说明：本接口返回数据量较多，查询时请控制每批次数量（<10）和适当减少查询字段数(<50)

💡 
 该接口会按照应用拥有的「待入职人员」的权限范围返回数据，请提前在「开发者后台 - 权限管理 - 数据权限-飞书人事(企业版)数据权限范围」中申请「待入职人员」权限范围

### 查询参数

| 参数名 | 类型 | 必填 | 描述 |
| ------ | ---- | ---- | ---- |
| page_size | integer | 是 | 分页大小，最大 10 |
| page_token | string | 否 | 分页标记，第一次请求不填，表示从头开始遍历；分页查询结果还有更多项时会同时返回新的 page_token，下次遍历可采用该 page_token 获取查询结果 |
| user_id_type | string | 否 | 用户 ID 类型 |
| department_id_type | string | 否 | 此次调用中使用的部门 ID 类型 |
### 请求体

| 参数名 | 类型 | 必填 | 描述 |
| ------ | ---- | ---- | ---- |
| pre_hire_ids | undefined[] | 否 | 待入职人员 ID 列表 <br> **示例**:  <br> **数据校验规则**:<br>最大长度: 10最小长度: 0 |
| fields | undefined[] | 否 | 返回数据的字段列表，填写方式：<br>- 为空时只返回 pre_hire_id<br>- 不为空时按照传入的字段返回数据，数据结构可以参考response的数据结构，格式示例如下：<br>    - person_info（个人信息） 字段：person_info.gender，person_info.age<br>    - employment_info（雇佣信息） 字段：employment_info.department<br>    - onboarding_info（入职信息） 字段：onboarding_info.onboarding_date<br>    - probation_info（试用期信息） 字段：probation_info.probation_period<br>    - contract_info（合同信息） 字段：contract_info.contract_type<br>- 如果要返回所有下级，只用传上级结构体名称，例如 person_info<br>- 返回数据越多，查询接口性能越慢，请按需填写返回字段 <br> **示例**: \["person_info","employment_info.department","probation_info.probation_period"\] <br> **数据校验规则**:<br>最大长度: 100最小长度: 0 |

**请求体示例**:

```json
{
    "pre_hire_ids": [
        "7094136522860922112"
    ],
    "fields": [
        "employment_info.department"
    ]
}
```
